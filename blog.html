What is the Document Object Model (DOM)?

The Document Object Model (DOM) is a programming interface for web documents. It represents the structure of an HTML or XML document as a tree of objects, where each part of the document (such as elements, attributes, and text) is represented as a node.

What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique in the DOM where a single event listener is attached to a parent element to manage events triggered by its child elements. Instead of adding individual event listeners to each child element, you can place a single listener on the parent and handle events as they "bubble up" through the DOM hierarchy.

How do you manipulate an element's attributes and styles using the DOM?

To get an attribute, use getAttribute(), and to set or remove one, use setAttribute() or removeAttribute(). For standard attributes, you can access them directly (e.g., element.id). For styles, you can retrieve inline styles with element.style or get computed styles using getComputedStyle(). To set styles, use the style property (e.g., element.style.color = 'red'), or apply multiple styles with cssText. Additionally, use classList to manage CSS classes (e.g., element.classList.add('active')). This approach allows for dynamic and efficient manipulation of the DOM.
